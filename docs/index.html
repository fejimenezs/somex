El problema que resolvemos

¿Por qué ya no más hojas de cálculo?

Cuando el soporte técnico se gestiona en archivos dispersos nadie sabe qué está abierto, qué tan rápido se resuelve, ni quién tiene más carga.

  • 📧
    Sin el sistema El usuario manda un correo, queda en el buzón, nadie sabe si se resolvió.
  • 📊
    Sin el sistema Cada analista tiene su lista en Excel. No hay visibilidad compartida.
  • 🚨
    Sin el sistema Gerencia no sabe cuántos casos urgentes llevan más de 24h sin respuesta.
  • ✅
    Con el Gestor TIC SOMEX Todo queda registrado, asignado, rastreado y medido automáticamente.

Lo que gana SOMEX con este sistema

  • Cero solicitudes perdidas — cada reporte queda registrado con fecha y hora
  • Asignación clara — cada caso va a un analista específico según su especialidad
  • Historial completo — cada cambio queda en la bitácora para auditoría
  • Alertas automáticas — los casos urgentes no pueden quedarse sin atención
  • Reportes para la gerencia — KPIs actualizados sin hacer nada extra
  • Cumplimiento del SLA — el sistema mide si se están respetando los tiempos

El flujo paso a paso

¿Cómo funciona?

Desde que un usuario reporta un problema hasta que queda solucionado, el sistema lleva el registro completo de cada paso.

📝

1. Usuario crea la solicitud

Describe el problema, elige la categoría y la prioridad. Solo toma 2 minutos.

Abierta
🔧

2. Analista la recibe

El técnico TIC asignado ve la solicitud en su lista y empieza a trabajar.

En Proceso
💡

3. Problema resuelto

El analista aplica la solución y registra qué se hizo. El usuario puede verificar.

Resuelta
🔒

4. Solicitud cerrada

El caso se archiva en el historial permanente. No puede reabrirse ni modificarse.

Cerrada

→

🟡 En Proceso

El analista ya la está atendiendo.

→

🟢 Resuelta

Solución aplicada, pendiente confirmar.

→

⚫ Cerrada

Confirmada y archivada definitivamente.

¿Qué puede hacer el sistema?

Funcionalidades principales

Todo lo que el equipo TIC de SOMEX necesita para trabajar de forma ordenada y medible.

📋

Registro de solicitudes

El usuario llena un formulario sencillo y el sistema lo registra automáticamente con fecha y hora.

  • Título y descripción del problema
  • Tipo: Incidente / Requerimiento / Mejora
  • Prioridad: Alta / Media / Baja
  • Quién lo reporta y desde qué área
🔍

Búsqueda y filtros

Encuentra cualquier solicitud en segundos con filtros inteligentes.

  • Filtrar por estado actual
  • Filtrar por prioridad
  • Filtrar por rango de fechas
  • Filtrar por analista o usuario
👤

Asignación a analistas

Cada solicitud se asigna a un analista según su especialidad técnica.

  • Catálogo de analistas disponibles
  • Reasignación cuando sea necesario
  • Visibilidad de carga por persona
📜

Historial completo

Cada cambio queda registrado con fecha, hora y quién lo realizó.

  • Quién cambió el estado y cuándo
  • Cuánto tiempo tardó cada etapa
  • Registro permanente e inalterable
👥

Gestión de personas

Administra el catálogo de usuarios solicitantes y analistas del sistema.

  • Agregar / editar / eliminar usuarios
  • Agregar / editar / eliminar analistas
  • Datos de contacto y área
🛡️

Control y seguridad

Reglas automáticas para que el flujo de trabajo siempre se respete.

  • Una solicitud cerrada no puede reabrirse
  • Validación de campos obligatorios
  • Confirmación antes de eliminar

Alertas automáticas de SLA

¿Qué pasa cuando algo urgente lleva demasiado tiempo?

🚨

Alerta de incumplimiento de SLA

El SLA (Service Level Agreement) de SOMEX establece que las solicitudes de prioridad Alta deben atenderse en menos de 24 horas. Si eso no ocurre, el sistema actúa solo:

  • Cada mañana a las 7:00 a.m. el sistema detecta automáticamente los casos urgentes sin atender por más de 24 horas.
  • Los registra en una lista especial de alertas con nombre del usuario, analista y horas transcurridas.
  • Power Automate envía una notificación al equipo (correo, Teams o SharePoint) sin que nadie tenga que hacer nada.
  • El equipo puede actuar de inmediato y el indicador de cumplimiento SLA queda registrado en el dashboard.
