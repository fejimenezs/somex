El problema que resuelve

¿Por qué ya no más hojas de cálculo?

Cuando el soporte técnico se gestiona en archivos de Excel dispersos, nadie sabe realmente cuántos problemas hay abiertos, qué tan rápido se están resolviendo, ni quién tiene demasiada carga de trabajo.

El Gestor de Solicitudes TIC reemplaza ese caos con una aplicación web ordenada donde los usuarios reportan sus problemas y el equipo técnico los atiende con trazabilidad completa.

😤
Sin el sistema El usuario manda un correo, queda en el buzón, nadie sabe si se resolvió.
📅
Sin el sistema Cada analista tiene su lista personal en Excel. No hay visibilidad compartida.
📈
Sin el sistema El gerente no puede saber cuántos problemas urgentes llevan más de un día sin respuesta.
✅
Con el Gestor TIC Todo queda registrado, asignado, rastreado y medido automáticamente.

El flujo paso a paso

¿Cómo funciona?

Desde que un usuario reporta un problema hasta que queda solucionado, el sistema lleva el registro completo de cada paso.

📝

1. Usuario crea la solicitud

Describe el problema, elige la categoría (Incidente, Requerimiento o Mejora) y la prioridad.

Abierta
🔧

2. Analista la recibe

Un analista del equipo TIC es asignado y comienza a trabajar en la solución.

En Proceso
💡

3. Problema resuelto

El analista aplica la solución y cambia el estado. El usuario puede verificar.

Resuelta
🔒

4. Solicitud cerrada

Cuando el usuario confirma, la solicitud se cierra y queda en el historial permanente.

¿Qué puede hacer el sistema?

Funcionalidades principales

Todo lo que necesita un equipo de soporte técnico para trabajar de forma ordenada y medible.

📋

Registro de solicitudes

El usuario llena un formulario sencillo y el sistema lo registra automáticamente.

  • Título y descripción del problema
  • Tipo: Incidente / Requerimiento / Mejora
  • Prioridad: Alta / Media / Baja
  • Quién lo reporta y desde qué área
🔍

Búsqueda y filtros

Encuentra cualquier solicitud en segundos con filtros inteligentes.

  • Filtrar por estado actual
  • Filtrar por prioridad
  • Filtrar por rango de fechas
  • Buscar por usuario o analista
👤

Asignación a analistas

Cada solicitud se puede asignar a un analista específico según su especialidad.

  • Catálogo de analistas disponibles
  • Reasignación cuando sea necesario
  • Visibilidad de la carga por persona
📜

Historial completo

Cada cambio queda registrado con fecha, hora y quién lo hizo.

  • Quién cambió el estado y cuándo
  • Cuánto tiempo tardó cada etapa
  • Registro permanente e inalterable
👥

Gestión de personas

Administra el catálogo de usuarios y analistas del sistema.

  • Agregar / editar / eliminar usuarios
  • Agregar / editar / eliminar analistas
  • Datos de contacto y área
🛡️

Control y seguridad

Reglas automáticas para que el flujo de trabajo se respete siempre.

  • Una solicitud cerrada no puede reabrirse
  • Validación de campos obligatorios
  • Confirmación antes de eliminar

Alertas automáticas de SLA

¿Qué pasa cuando algo urgente lleva demasiado tiempo?

🚨

Alerta de incumplimiento de SLA

El SLA (Service Level Agreement) es el acuerdo de que las solicitudes de prioridad Alta deben atenderse en menos de 24 horas. Si eso no ocurre, el sistema actúa automáticamente:

  • Cada mañana a las 7:00 a.m., el sistema busca solicitudes urgentes sin atender por más de 24 horas.
  • Las registra en una lista especial de alertas.
  • Power Automate envía una notificación automática al equipo (correo, Teams o SharePoint).
  • El equipo puede actuar de inmediato sin depender de que alguien revise manualmente.

¿Para quién es este sistema?

Tres tipos de personas lo usan

👩‍💼

El Usuario Solicitante

Cualquier empleado de la empresa que tenga un problema técnico. No necesita saber de tecnología — solo llena un formulario sencillo y el sistema hace el resto.

🧑‍💻

El Analista TIC

El técnico de soporte que recibe las solicitudes asignadas. Ve su lista de tareas, actualiza el estado y registra cómo resolvió cada caso.

👨‍💼

El Gerente / Director

El jefe del área que necesita saber si el equipo está respondiendo bien. Ve el dashboard de Power BI con todos los indicadores sin entrar al sistema.

Tecnología detrás del sistema

No necesitas entender esto para usar el sistema — pero aquí está el detalle para quienes les interese saber cómo funciona por dentro.

🌐

El sitio web de la aplicación, lo que ves y usas.

Los gráficos y tableros que muestran cómo está funcionando el soporte.

🎨

Bootstrap 5

El diseño responsivo que hace que el sistema se vea bien en cualquier pantalla.

¿Cómo se conectan todas las piezas?

Los usuarios interactúan con la aplicación web → los datos se guardan en SQL Server → un Job automático detecta incumplimientos de SLA cada mañana → Power Automate envía notificaciones → Power BI muestra los datos actualizados para que gerencia tome decisiones.
